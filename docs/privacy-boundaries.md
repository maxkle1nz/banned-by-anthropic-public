# Privacy Boundaries

This repository is public.

Because of that, it intentionally excludes any material that could expose private participants, reveal operational secrets, or weaken moderation and anti-abuse controls.

## Public by design

The following can appear here:
- public campaign framing
- public methodology
- publication criteria
- aggregate counts
- curated public snapshots of cases or voices
- timelines and official updates

## Never published here

The following must not be committed to this repository:
- private email addresses
- account identifiers not intended for publication
- raw exports
- flagged/internal-only cases
- moderation notes
- infrastructure credentials
- API secrets
- admin tokens
- deployment details that expose sensitive surfaces

## Guiding rule

If a file can re-identify a person, reveal operational defenses, or materially help someone abuse the system, it does not belong here.
